You are performing a KashFlow ‚Üí Supabase customer delta sync for the Brainbox Candy Trade CRM. Work silently and efficiently. Report only the final summary.

# Context

- **Source table: `brainbox_candy.five_minute_customers`** ‚Äî a cleaned, geocoded mirror maintained by Dave's pipeline. Updated nightly around 20:00 UTC. This is the canonical CRM source (NOT the raw `customers` table).

# Name-prefix conventions (CRITICAL ‚Äî read carefully)

The KashFlow team uses two name prefixes as control signals. Respect them exactly.

- **Single asterisk `*` (e.g. `*Acme Cards`)** = customer has ceased trading. Sync the record but force `hidden=true, status='inactive'`. Keep the `*` prefix in `company_name` so it's still identifiable if someone shows hidden customers.
- **Double asterisk `**` (e.g. `**Acme Cards`)** = active territorial protection. Sync normally. Set `territorial_protection=true`. **KEEP the `**` prefix in `company_name`** ‚Äî this is how the user identifies them at a glance when sorting alphabetically. Do NOT strip it.
- **No prefix** = normal customer. Sync normally. `territorial_protection=false`.

Detection rules in SQL (MariaDB):
- `single_asterisk = Name LIKE '*%' AND Name NOT LIKE '**%'`
- `double_asterisk = Name LIKE '**%'`

# CRM-OWNED columns ‚Äî NEVER touch these

`pipeline_stage, assigned_to, tags, mailerlite_id, last_contacted_at, ml_any_subscribed, ml_group_names, mailerlite_subscribed, mailerlite_status, mailerlite_subscriber_id, mailerlite_group_names, mailerlite_refreshed_at, shrig_only, new_non_ha, naked, notes, campaign_*`

# Routine-managed columns (set from Name prefix only)

- `hidden` ‚Äî set to `true` only when `single_asterisk`; otherwise **leave whatever value it had**
- `status` ‚Äî set to `'inactive'` only when `single_asterisk`; otherwise **leave whatever value it had** (new inserts get `'prospect'`)
- `territorial_protection` ‚Äî always derived from the current Name (`double_asterisk` -> true, else false). Always sync this on every row.

# Steps

## Step 1 ‚Äî Determine sync mode

Read Supabase `sync_state`:
- key=`kashflow_customer_last_sync`
- key=`kashflow_customer_last_full_sweep`

- If `last_sync IS NULL` -> **full sweep** (first run ever)
- If today is Sunday AND `last_full_sweep < today's date` -> **full sweep** (weekly refresh)
- Otherwise -> **delta**

## Step 2 ‚Äî Pull MySQL data

- **Delta mode**: `WHERE last_updated >= '<last_sync - 1 day>'`
- **Full sweep mode**: no WHERE clause (all ~7,077 rows)

Join to `customer_sources` for source name:
```sql
SELECT
  fmc.*,
  cs.Name AS source_name
FROM five_minute_customers fmc
LEFT JOIN customer_sources cs ON cs.ID = fmc.Source
WHERE <delta clause as above>
ORDER BY fmc.Code;
```

For full sweeps, page through in batches of ~200 rows to avoid timeouts.

## Step 3 ‚Äî Fetch existing Supabase rows

For every `Code` value returned in step 2, fetch the matching Supabase row (by `kf_code`). Use IN(...) in chunks of 500.

Select these columns for comparison: `kf_code, kf_customer_id, company_name, address_1, address_2, address_3, address_4, postcode, country_name, phone, mobile, website, source, discount, payment_terms, store_type, vat_number, eori, member_number, po_number, delivery_address_1, delivery_address_2, delivery_address_3, delivery_address_4, delivery_postcode, kf_created_date, lat, lng, formatted_address, territorial_protection, hidden, status`

## Step 4 ‚Äî Compare and apply

For each MySQL row:

**Determine prefix flags:**
- `single_asterisk = Name starts with '*' but not '**'`
- `double_asterisk = Name starts with '**'`

**If no matching Supabase row** (new insert):
- Build full payload with all synced columns
- `company_name = Name` (preserve prefix)
- `territorial_protection = double_asterisk`
- If `single_asterisk`: also set `hidden=true, status='inactive'`
- Else: `hidden=false, status='prospect'`
- Write `kf_synced_at = now(), created_at = now(), updated_at = now()`

**If matching row exists** (update):
- Compare each KashFlow-owned column (see mapping table)
- Always compare and potentially update `territorial_protection` (derived fresh from Name)
- For `hidden` and `status`: only write them if `single_asterisk` is true (force to true/'inactive'). Otherwise leave whatever they were.
- For `customer_notes`: skip entirely on updates (CRM may have edited).
- If any column differs, build UPDATE payload with ONLY changed columns + `updated_at = now()`.
- If no differences -> skip (counts as unchanged).

## Step 5 ‚Äî Log the run

INSERT into `customer_sync_log`:
- `triggered_by`: 'routine' (or the value passed in by the caller if provided)
- `filename`: `'kashflow-delta-' || sync_mode || '-' || now_iso`
- `total_rows`: count of MySQL rows considered
- `created_count`, `updated_count`, `unchanged_count`, `error_count`
- `duration_ms`
- `changes_summary`: array of `{kf_code, changed_fields:[...]}` ‚Äî cap at 200 entries
- `errors`: array ‚Äî cap at 50

## Step 6 ‚Äî Update sync_state

```sql
UPDATE sync_state SET value_ts = now(), updated_at = now() WHERE key = 'kashflow_customer_last_sync';
```

If this was a full sweep, also update `'kashflow_customer_last_full_sweep'`.

# Error handling

- If MySQL is unreachable: log to `customer_sync_log` with `error_count=0` and `errors=[{top_level:"MySQL unreachable"}]`, do NOT update `sync_state`. Return the error clearly.
- If any individual row fails to write: record in `errors` array, continue with the rest. Only treat as total failure if >10% of rows error.
- Never retry destructive operations. Log and move on.
